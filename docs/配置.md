# 配置参考

Portal 通过单个 URL 进行配置。

```text
portal://<shared-key>@<listen-host>:<listen-port>?tls=<mode>&spec=<spec>&alpn=<alpn>&net=<mode>&dial=<ip-or-auto>&rate=<mbps>&etar=<mbps>&crt=<path>&key=<path>&log=<level>
```

URL 中的用户名即为共享密钥（shared key）。不支持密码部分。必须指定监听端口和非空的共享密钥。未知的查询参数将被忽略。

对于共享密钥、`spec`、`alpn` 及文件路径中包含的 URL 保留字符，请使用百分号编码（percent encoding）。

## 输入规则

共享密钥、`spec` 和 `alpn` 会按 UTF-8 格式进行百分号解码。`spec` 或 `alpn` 中的字面字符 `+` 保持为 `+`，不会被转换为空格。如果查询参数的键（key）出现多次，则仅使用第一次出现的值。

| 输入项   | 要求               | 解码后的 UTF-8 字节长度 |
| -------- | ------------------ | ----------------------- |
| 共享密钥 | 必需且非空         | `1..255`                |
| `spec`   | 可选；为空表示省略 | 非空时为 `1..255`       |
| `alpn`   | 可选；为空表示省略 | 非空时为 `1..255`       |

## 参数

| 参数   | 默认值  | 语义                                                         |
| ------ | ------- | ------------------------------------------------------------ |
| `tls`  | `1`     | `1` 表示生成内存中的自签名证书。`2` 表示从 `crt` 和 `key` 指定的路径加载 PEM 文件。`0` 及其他所有值均会被拒绝。 |
| `spec` | `auto`  | 用于 v1 认证材料、确定性填充（deterministic padding）及字段顺序的种子值。 |
| `alpn` | `now/1` | TLS 和 QUIC 的 ALPN 值。它不会改变认证、填充或帧布局。       |
| `net`  | `mix`   | 选择入站传输协议：`tcp` 启用 TLS/TCP，`udp` 启用 QUIC/UDP，`mix` 同时启用两者。若未设置或值为空，则默认为 `mix`。 |
| `dial` | `auto`  | 用于出站 TCP 和 UDP 套接字的本地 IP 地址。若值为空、无效、为主机名或设为 `auto`，则使用操作系统默认值。 |
| `rate` | `0`     | 客户端到目标端的流量限制（单位：Mbps）。                     |
| `etar` | `0`     | 目标端到客户端的流量限制（单位：Mbps）。                     |
| `crt`  | 空      | `tls=2` 模式下使用的 PEM 证书链。                            |
| `key`  | 空      | `tls=2` 模式下使用的 PEM 私钥。                              |
| `log`  | `info`  | 日志级别：`none`、`debug`、`info`、`warn`、`error` 或 `event`。若值为未知，则默认为 `info`。 |

`rate` 和 `etar` 接受正十进制整数。若值为零、负数、无效值或未设置，则禁用相应方向的流量限制。换算公式如下：

```text
bytes_per_second = mbps * 125000
```

## 传输能力

`net` 参数用于选择监听端的传输协议，它并不直接决定被代理流量的类型。

| 监听传输协议         | TCP 代理流量                    | UDP 代理流量                      |
| -------------------- | ------------------------------- | --------------------------------- |
| TLS/TCP (`net=tcp`)  | 每个已认证连接对应一个 TCP 转发 | 每个已认证连接对应一个 UoT 数据流 |
| QUIC/UDP (`net=udp`) | 每个双向流对应一个 TCP 转发     | 多路复用的 QUIC DATAGRAM 数据流   |
| 两者 (`net=mix`)     | 上述两种路径                    | 上述两种路径                      |

UoT 没有单独的 Portal 设置。兼容的客户端在已认证的 TLS/TCP 连接内，通过使用预留的 UoT 请求目标地址，并发送带有长度前缀的 UDP 数据包来启用该功能。关于传输格式，请参阅 [协议规范](protocol.md#92-udp-over-tcp-uot)。

## 监听地址规则

若监听主机部分为空，则针对每个选定的传输协议，在同一端口上分别绑定 IPv4 和 IPv6 的通配符套接字：

```text
portal://secret@:2077
```

如有需要，可明确指定绑定特定的地址族：

```text
portal://secret@0.0.0.0:2077
portal://secret@[::]:2077
```

使用 IP 字面量时，将绑定该特定地址。使用主机名时，系统会进行解析并使用解析出的第一个地址。在 Portal 开始接收流量之前，所有选定的套接字必须均已成功绑定。在 `net=mix` 模式下，若 TCP 或 UDP 中任一绑定失败，则启动失败。

## TLS 模式

`tls=1` 模式下，Portal 启动时会为 `localhost` 生成一个新的自签名证书：

```text
portal://secret@:2077?tls=1
```

客户端必须明确信任或固定（pin）此模式。生成的证书在重启后不会保持不变。

`tls=2` 模式用于加载 PEM 格式的证书链和私钥：

```text
portal://secret@:2077?tls=2&crt=/etc/nowhere/cert.pem&key=/etc/nowhere/key.pem
```

这两个文件在启动时均必须有效。TLS/TCP 和 QUIC 将使用相同的证书和 ALPN。

不支持明文模式（即 `tls=0`）。

## Spec 与 ALPN

`spec` 和 `alpn` 是两个独立的配置项。

```text
effective_spec = 若第一个 `spec` 值非空则使用解码后的该值，否则为 "auto"
effective_alpn = 若第一个 `alpn` 值非空则使用解码后的该值，否则为 "now/1"
```

更改 `spec` 会改变 v1 认证常量、确定性填充（deterministic padding）以及字段顺序。更改 `alpn` 仅影响 TLS 和 QUIC 协商过程。通信双方必须就这两个值达成一致，才能实现互操作。

## 出站源地址

`dial` 参数可选择性地将出站套接字绑定到特定的本地 IP 地址：

```text
portal://secret@:2077?dial=192.0.2.10
portal://secret@:2077?dial=2001:db8::10
```

仅接受 IP 字面量。若设置为 `auto`、空值、主机名或无效地址，则由操作系统选择源地址。一旦指定了 IP，Portal 将仅处理属于同一地址族（address family）的目标地址。

## 日志级别

默认级别为 `info`：

```text
portal://secret@:2077?log=info
```

若仅需输出机器可读的检查点（checkpoint）记录，请使用 `event`：

```text
portal://secret@:2077?log=event
```

仅当由其他监控程序（supervisor）负责捕获就绪状态和故障状态时，才使用 `none`。

## 示例

使用默认配置的双栈混合服务：

```text
portal://secret@:2077
```

仅限 TLS/TCP 的入口（包含 TCP 转发和 UoT）：

```text
portal://secret@:2077?net=tcp
```

仅限 QUIC 的入口（包含流转发和 DATAGRAM UDP）：

```text
portal://secret@:2077?net=udp
```

使用 PEM 证书并启用事件日志：

```text
portal://secret@:2077?tls=2&crt=/etc/nowhere/cert.pem&key=/etc/nowhere/key.pem&log=event
```

方向性速率限制：

```text
portal://secret@:2077?rate=100&etar=200
```

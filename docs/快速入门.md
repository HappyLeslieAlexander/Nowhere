# 快速入门

本指南介绍如何从源码构建并启动 Nowhere v1 Portal。

## 要求

- 支持 Rust 2024 edition 的 Rust 工具链。
- 一台能够绑定所选 TCP 和/或 UDP 监听端口的主机。
- 一个支持 Nowhere v1 协议的客户端实现。

## 构建

```bash
cargo build --release --locked
```

生成的 Release 模式二进制文件位于：

```text
target/release/nowhere
```

查看命令行选项：

```bash
./target/release/nowhere --help
./target/release/nowhere --version
```

## 启动本地 Portal

```bash
./target/release/nowhere 'portal://secret@:2077'
```

这将使用以下默认配置启动 Portal：

| 设置项                 | 实际生效值                               |
| ---------------------- | ---------------------------------------- |
| 共享密钥 (shared key)  | `secret`                                 |
| 监听主机 (listen host) | 空主机名，双栈通配符地址                 |
| 监听端口 (listen port) | `2077`                                   |
| `tls`                  | `1`，生成自签名证书                      |
| `net`                  | `mix`，同时启用 TLS/TCP 和 QUIC/UDP 监听 |
| `spec`                 | `auto`                                   |
| `alpn`                 | `now/1`                                  |
| `dial`                 | `auto`                                   |
| `rate`                 | `0`，已禁用                              |
| `etar`                 | `0`，已禁用                              |
| `log`                  | `info`                                   |

启动日志会输出解析后的实际 URL。请将该行视为当前运行进程的权威配置信息（供运维人员参考）。 ## 绑定特定地址

仅绑定 IPv4：

```bash
./target/release/nowhere 'portal://secret@0.0.0.0:2077'
```

仅绑定 IPv6：

```bash
./target/release/nowhere 'portal://secret@[::]:2077'
```

若只需监听单一传输协议，可使用 `net=tcp` 或 `net=udp`：

```bash
./target/release/nowhere 'portal://secret@:2077?net=tcp'
./target/release/nowhere 'portal://secret@:2077?net=udp'
```

这些参数指定的是入口传输协议，而非代理的负载类型。
`net=tcp` 承载 TCP 转发和 UDP-over-TCP (UoT) 流量；`net=udp` 则通过 QUIC 双向流承载 TCP 转发，并通过 QUIC DATAGRAM 帧承载 UDP 流量。

## 使用 PEM 证书

默认的 `tls=1` 模式会在启动时生成新的自签名证书。若需从磁盘加载证书链和私钥，请使用 `tls=2`：

```bash
./target/release/nowhere 'portal://secret@:2077?tls=2&crt=/etc/nowhere/cert.pem&key=/etc/nowhere/key.pem'
```

启动时这两个文件均须有效。程序会在处理 `ClientHello` 且距离上次加载已超过 `NOW_RELOAD_INTERVAL` 时，尝试重新加载证书。

## 停止运行

按下 `Ctrl-C` 发送 `SIGINT` 信号。Portal 将关闭监听器和活动连接，等待 QUIC 端点进入空闲状态（最长等待时间为 `NOW_SHUTDOWN_TIMEOUT`），重置活动速率限制器，刷新日志缓冲区，随后退出。
